Each markdown header becomes a lowercase XML tag. Spaces and special characters are replaced with underscores. Headers are only indented when nested inside other headers, not based on their markdown level.

## ROLE
You are a helpful assistant

becomes:

<role>
  You are a helpful assistant
</role>

Italic/Bold Headings

Lines with italic (_Section_) or bold (__SECTION__) formatting at the root level (no indentation) are treated as level 1 headers. Note: Only underscores (_) create headers, not asterisks (*).

__CONTRAINTS__
- Do not do x

becomes:

<contraints>
  <item>Do not do x</item>
</contraints>

Nesting

Smaller headers are nested inside larger ones based on heading level. Headers at the root level (no parent header) are not indented, regardless of their markdown level.

# PERSONALITY
## Tone
Be polite

becomes:

<personality>
  <tone>
    Be polite
  </tone>
</personality>

Lists

Bullet lists (- item) become plain <item> tags.

Numbered lists (1. item) include a number attribute.

Nested lists are properly indented inside parent list items. Indent nested list items with spaces.

- When y do:
  1. First step
  2. Second step

becomes:

<item>When y do:
  <item number="1">First step</item>
  <item number="2">Second step</item>
</item>

Code Blocks

Fenced code blocks are wrapped in <code lang="..."> tags with CDATA. The language is detected from the code fence identifier.

# EXAMPLE
```python
print("Hey")
```

becomes:

<example>
  <code lang="python"><![CDATA[
print("Hey")
]]></code>
</example>
